# UI 手動確認チェックリスト

本番リリース前または大きな変更後は、以下のシナリオに沿って手動確認を実施してください。  
コマンドは Docker コンテナまたはローカル環境で `scripts/run_all_services.sh` もしくは `docker-compose up` を利用して起動した状態を前提とします。

## 1. ログインフロー
- `/` にアクセスしログイン画面へリダイレクトされることを確認。
- 正しい認証情報でログインし、ダッシュボード（アップロード画面）へ遷移すること。
- 誤ったパスワードでエラー表示がされ、CSRF トークン未設定時は 403 になること。
- `/login/?next=/gallery/` からログインし、成功後 `next` 先へ遷移すること。

## 2. アップロードと変換
- JPEG / PNG / WebP を 1 枚以上ドロップし、プレビューが表示されること。
- 10MB 超のファイル、未サポート拡張子が拒否されること。
- プリセットプロンプト選択・カスタム入力が反映されること。
- 変換開始ボタン押下後、処理中ページへ遷移し、X/Y 枚表示と進捗バーが更新されること。
- キャンセルボタンで `/` に戻り、通知が表示されること。

## 3. 処理完了後のギャラリー
- 変換完了後、自動で `/gallery/` に遷移する（またはリンクから遷移し、最新の変換が先頭に表示される）こと。
- 検索／並び替え／表示件数変更が即時反映されること。
- 各カードから詳細モーダルを開き、Before/After・生成日時・プロンプトが表示されること。
- 輝度調整が意図した明るさに変化すること、リセットが効くこと。
- ダウンロードリンクで画像が保存できること。
- 削除ボタンでカードが非表示になり、一覧を再読み込みすると反映されること。

## 4. 利用状況確認
- ヘッダーの利用状況がログイン直後・変換直後で即時更新されること。
- `/api/v1/usage/` を直接叩き、キャッシュが最新値を返すこと（変換→再取得）。

## 5. レスポンシブ表示
- スマートフォン幅（≤576px）、タブレット幅（≤768px）、PC 幅でレイアウトが崩れないこと。
- トースト通知が画面幅に合わせて適切に表示されること。

各項目の確認結果を記録し、問題があればチケット化してください。
